# מתכנן אירועים — HANAN APP

אפליקציית ווב לתכנון והדמיה ויזואלית של אירועים: מסדרים את האולם בעורך דו־ממדי מקצועי בסגנון CAD, ורואים את התוצאה **מיידית** בתצוגה תלת־ממדית חיה — שתיהן ניזונות מאותו מודל סצנה אחד.

הממשק כולו בעברית (RTL), והמוצר מיועד גם למתכנני אירועים מקצועיים וגם ללקוחות פרטיים.

## מה יש בפנים

- **דשבורד פרויקטים** — יצירת אירוע (שם, תאריך, מידות אולם, התחלה ריקה או פריסת דוגמה), כרטיסים עם תמונת תצוגה מקדימה חיה של התוכנית, מחיקה עם אישור.
- **עורך 2D** — קנבס מדויק עם רשת אדפטיבית והצמדה, גרירה עם קווי יישור ומחווני מרחק מהקירות, סיבוב ושינוי גודל בידיות, בחירה מרובה (marquee ,Shift), שכפול/העתקה/הדבקה, תפריט הקשר, ביטול/ביצוע חוזר ברמת מחווה, זום אל הסמן ופאן.
- **ספריית אובייקטים מונחית־דאטה** — שולחנות (עגול, מלבני, מרובע, אבירים, קוקטייל), כיסאות, במה, רחבת ריקודים, בר, עמדת DJ, בופה, צמחים ומחיצות. הוספת פריט חדש = קובץ קטלוג בלבד, בלי לגעת ברנדררים.
- **הושבה אוטומטית** — בוחרים כמות מקומות והכיסאות מסתדרים סביב השולחן; הם עוקבים אחרי הזזה, סיבוב ושינוי גודל. דאבל־קליק על כיסא בודד מאפשר לכוון אותו ידנית או לנתק אותו.
- **תצוגת 3D חיה** — האולם עם קירות, תאורה וצללים; כל שינוי ב־2D (מיקום, צבע מפה, כמות כיסאות, בחירה) משתקף מיידית; שלושה מצבי מצלמה (מבט כללי, מבט עליון, גובה עיניים) עם מעברים חלקים; בחירת אובייקט בקליק מסונכרנת לשני הכיוונים.
- **מסך מפוצל** — עורכים ב־2D מימין ורואים את ה־3D משמאל, עם מחיצה נגררת.
- **שמירה אוטומטית** — הפרויקטים נשמרים מקומית (IndexedDB) עם חיווי שמירה; רענון או סגירה לא מאבדים כלום.
- **ייצוא** — תוכנית רצפה כ־PNG נקי (בלי רשת וסימוני עריכה, עם תוויות בעברית) וקובץ פרויקט JSON מלא הניתן לייבוא חזרה.
- **Inspector חכם** — עריכת מידות במטרים, סיבוב, צבעי מפה וכיסאות מתוך פלטת אירועים, מספור שולחנות, יישור ופיזור לבחירה מרובה, והגדרות אולם (מידות, גובה קירות, רצפה).
- **קיצורי מקלדת מלאים** — פועלים גם בפריסת מקלדת עברית (קישור למקשים פיזיים). לחצו `?` בתוך העורך לרשימה המלאה.

## טכנולוגיות

| שכבה | בחירה |
|---|---|
| UI | React 19 · TypeScript (strict) · Vite 6 · Tailwind CSS v4 |
| עורך 2D | Konva / react-konva |
| תצוגת 3D | three.js · @react-three/fiber · drei |
| ניהול מצב | Zustand 5 + Immer, ‏undo/redo עם zundo |
| התמדה | IndexedDB (idb) עם סכמה מגורסת, מיגרציות וולידציית zod |
| טיפוגרפיה | Assistant (עברית) · IBM Plex Mono (מספרים ומידות) |

**עיקרון הארכיטקטורה:** מודל סצנה אחד (`src/core`, ללא תלות ב־React/Konva/three) הוא מקור האמת היחיד. עורך ה־2D ותצוגת ה־3D הם שני "מפרשים" של אותם נתונים — אין המרות כפולות ואין מצב משוכפל. במהלך גרירה העדכונים ל־3D עוברים במסלול transient (מוטציה ישירה של אובייקט three ללא רינדור React), כך שהסנכרון נשאר חלק גם עם מאות כיסאות.

```
src/
  core/         מודל הסצנה, קטלוג האובייקטים, גאומטריית הושבה והצמדה, מיגרציות
  state/        Zustand store, פעולות (מסלול הכתיבה היחיד), סלקטורים
  editor2d/     קנבס Konva: שכבות, גרירה, Transformer, קיצורים, ייצוא תמונה
  viewer3d/     סצנת three: בניית mesh מהקטלוג, instancing לכיסאות, מצלמות, תאורה
  ui/           סרגל כלים, ספרייה, Inspector, שורת סטטוס, דיאלוגים (עברית, RTL)
  persistence/  מאגר פרויקטים (IndexedDB), שמירה אוטומטית, ייצוא/ייבוא
  app/          דשבורד ומעטפת האפליקציה
```

## התקנה והפעלה

דרישה מקדימה: **Node.js 20 ומעלה** (כולל npm).

```bash
git clone https://github.com/MSA-I/HANAN-APP.git
cd HANAN-APP
npm install
npm run dev
```

ואז פותחים דפדפן בכתובת **http://localhost:3000**, לוחצים "אירוע חדש" — מומלץ לבחור **"פריסה לדוגמה"** כדי לקבל אולם מאובזר ולהתחיל לשחק.

### פקודות נוספות

| פקודה | מה היא עושה |
|---|---|
| `npm run dev` | שרת פיתוח עם רענון חם (פורט 3000) |
| `npm run build` | בדיקת טיפוסים + בניית גרסת ייצור לתיקיית `dist/` |
| `npm run preview` | הגשה מקומית של גרסת הייצור |
| `npm test` | בדיקות יחידה (vitest) לליבה: הושבה, הצמדה, undo, סריאליזציה |
| `npm run lint` | בדיקת eslint |

הנתונים נשמרים ב־IndexedDB של הדפדפן — אין צורך בשרת או בחשבון. גיבוי/העברה בין מחשבים נעשים דרך ייצוא/ייבוא JSON מתפריט הייצוא.
